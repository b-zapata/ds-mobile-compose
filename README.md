# ds-mobile-compose

Doomscrolling Intervention Study — repository and app.

## Repository structure

| Path | Purpose |
|------|---------|
| `SYSTEM_ARCHITECTURE.md` | System and study specification |
| `BUILD_PLAN.md` | Implementation phases |
| `CURSOR_RULES.md` | Rules for AI coding agents |
| `mobile-app/` | Android app (Kotlin, Jetpack Compose) |
| `server/` | Backend ingestion API (Phase 11) |
| `infrastructure/` | Cloud/DB config (Phase 12) |
| `prompts/` | Prompt library (Phase 8) |

## Getting started

1. Read `SYSTEM_ARCHITECTURE.md` and `BUILD_PLAN.md`.
2. Open `mobile-app/` in Android Studio for the first sync and build.
3. Implement phases in order per `BUILD_PLAN.md`.
